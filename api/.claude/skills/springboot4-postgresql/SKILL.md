# Skill: Spring Boot 4 + PostgreSQL — Central REST API

> **Scope:** `/api` (Agent 1)
> **Overrides:** Any older community skills for this domain

---

## Environment

- You MUST use **Spring Boot 4.0.5** and **Java 24** for all API code.
- Rely on **Jakarta EE 11** standards (`jakarta.*` namespace) for persistence, validation, security, and dependency injection.
- Leverage Java 24 language features where they improve clarity: records for DTOs, sealed interfaces for domain hierarchies, pattern matching in switch expressions, and virtual threads for concurrent request handling where appropriate.
- PostgreSQL driver version: **42.7.2**. All database interactions go through Spring Data JPA with Flyway-managed migrations.

## Encryption

- **Application-level encryption is mandatory** for all sensitive fields before they are written to PostgreSQL. Sensitive fields include: social security number, diagnoses, allergies, current medication, and medical history.
- Implement encryption using one of:
  - `@ColumnTransformer` with database-level encryption functions, or
  - **JPA `AttributeConverter`** that applies AES-256-GCM encryption/decryption transparently at the entity layer (preferred approach for portability).
- Encryption keys MUST be loaded from environment variables or a secrets manager. Never hardcode keys in source code or configuration files committed to version control.
- Every encrypted field must use a unique random IV (Initialization Vector) per write operation.

## Auditing

- **Hibernate Envers is required** to track all modifications to auditable entities. Every entity that stores patient, practitioner, or appointment data must be annotated with `@Audited`.
- Envers revision tables must be included in Flyway migrations (not auto-generated by Hibernate in production).
- Audit queries (retrieving historical versions) must use the Envers `AuditReader` API, never raw SQL against revision tables.
- Complement Envers with an application-level `audit_log` table for recording read-access events (Envers only tracks writes).

## Performance

- **N+1 queries are strictly prohibited.** All entity relationships loaded in a query must use explicit `JOIN FETCH` in JPQL or `@EntityGraph` annotations.
- Before writing any repository method that returns entities with associations, verify the fetch strategy and add `JOIN FETCH` clauses as needed.
- **Parameterized queries only.** Every query, whether JPQL, native SQL, or Criteria API, must use parameter binding (`:param` or `?1`). String concatenation in queries is forbidden under all circumstances.
- Enable Hibernate query logging in development (`spring.jpa.show-sql=true`, `hibernate.format_sql=true`) and review generated SQL for unnecessary joins or repeated queries.
- Use pagination (`Pageable`) for all list endpoints. Never return unbounded result sets.
