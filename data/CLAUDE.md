## 1. PROJECT DEFINITION

This directory contains the data engineering pipeline and NoSQL storage module of RehabiAPP. It ingests, transforms, and stores telemetry data generated by the Unity rehabilitation games (routed through the Spring Boot /api). It also prepares analytics-ready datasets for practitioner visualization in the desktop ERP.

This is the data engineering core of the project. The focus is on ETL best practices, efficient schema design for read-heavy analytical queries, and aggregation pipelines that produce meaningful rehabilitation progress metrics.

4. **Schema design:** MongoDB schemas (Mongoose) must be optimized for read-heavy analytical queries. Embed data that is always read together. Reference data that changes independently. Index fields used in aggregation pipelines.

5. **ETL discipline:** Every data transformation must be idempotent (safe to run multiple times with the same result). Document the input format, transformation logic, and output format for every pipeline stage.

6. **No direct external access:** This service is internal. It receives data from the Spring Boot /api via internal endpoints. External clients (mobile, games, desktop) never call this service directly.

## 4. ARCHITECTURE

```
src/
    |-- models/            Mongoose schemas and models (GameSession, PatientProgress, etc.)
    |-- routes/            Express route definitions (internal ingestion endpoints)
    |-- controllers/       Request handlers (validation, delegation to services)
    |-- services/          Business logic (data transformation, aggregation orchestration)
    |-- pipelines/         MongoDB aggregation pipelines (analytics queries)
    |-- utils/             Helpers, date formatters, data validators
    |-- config/            Database connection, environment config
```

Controllers are thin. All transformation logic lives in services/ and pipelines/.

## 5. MONGOOSE SCHEMA REFERENCE

Primary collection for game session telemetry:

```javascript
GameSession {
  patientDni: String (indexed),
  gameId: String (indexed),
  progressionLevel: Number (indexed, 1-4),
  sessionStart: Date,
  sessionEnd: Date,
  durationSeconds: Number,
  score: Number,
  repetitionsCompleted: Number,
  repetitionsTarget: Number,
  movementMetrics: {
    rangeOfMotionDegrees: Number,
    averageSpeed: Number,
    maxSpeed: Number
  },
  completed: Boolean,
  receivedAt: Date (server timestamp)
}
```

Derived collection for pre-aggregated patient progress:

```javascript
PatientProgress {
  patientDni: String (indexed),
  gameId: String,
  progressionLevel: Number,
  period: String (e.g. "2026-W10", "2026-03"),
  totalSessions: Number,
  averageScore: Number,
  averageDuration: Number,
  completionRate: Number,
  rangeOfMotionTrend: [Number],
  lastUpdated: Date
}
```
